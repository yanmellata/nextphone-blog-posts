# PHASE 2: OUTLINE PROMPT

**Goal:** Create detailed blueprint for writing phase with section structure, data placement, and content elements mapped

**Time:** ~1.5 hours
**Input:** Completed research document from Phase 1
**Output:** Detailed section-by-section outline ready for Phase 3 (Writing)

**Reference:** See `NEXTPHONE_FACTBASE.md` and `CONTENT_PRODUCTION_GUIDELINES.md`

---

## Overview

The outline phase transforms research into an actionable writing plan. A great outline:
- Maps the user journey from problem to solution
- Specifies exactly what goes in each section
- Plans where data, images, links, and CTAs go
- Ensures all table stakes topics + differentiators are covered

**Why outline?** Writing without an outline leads to rambling content, forgetting key topics, poor flow, missing data/CTAs, and hours of rewriting. With an outline, writing becomes fill-in-the-blanks.

## Step 2.3: Detailed Section-by-Section Outline (30 mins)

**Goal:** Create blueprint for writing phase with complete detail

**For EACH Section, Define:**
1. **H2 Title** (include keyword variants naturally)
2. **Purpose** (what question does this answer?)
3. **Word Count Target** (sum should equal total target)
4. **H3 Subsections** (3-5 per H2)
5. **Key Points** (bullets of what to cover)
6. **Data/Stats to Include** (from research doc + NEXTPHONE_FACTBASE.md)
7. **Examples to Use** (customer quotes, case studies, scenarios)
8. **Visuals Needed** (chart, screenshot, diagram - placeholder description)
9. **Links to Add** (internal + external with anchor text)

**Section Template:**
```markdown
## SECTION [#]: [H2 Title]

**Purpose:** [What question does this answer?]
**Word Count Target:** [X-Y words]

**H3 Subsections:**
1. [Subsection 1 title]
2. [Subsection 2 title]
3. [Subsection 3 title]

**Key Points to Cover:**
- [Point 1]
- [Point 2]
- [Point 3]

**Data/Stats to Include:**
- **Stat 1:** [e.g., "74.1% missed calls"] - Source: NextPhone factbase
- **Stat 2:** [e.g., "AI market growing 25% YoY"] - Source: Gartner 2024

**Examples/Quotes:**
- Example: [e.g., "HVAC contractor scenario - on roof, can't answer"]
- Quote: "[Verbatim customer quote from factbase]"

**Visual Needed:**
- Type: [e.g., "Bar chart showing % missed calls by industry"]
- Placement: After [which paragraph/subsection]
- Alt text: "[Description]"

**Links to Add:**
- **Internal:** Link to [page] with anchor text "[text]" - Context: [why linking]
- **External:** Cite [source] with anchor text "[text]" - Context: [why citing]
```

**Typical blog structure (8-12 sections):**
1. Key Takeaways Box (3-6 bullets)
2. Intro (100-150 words)
3. Section 1: Context/Background (300-400 words)
4. Section 2: Problem Deep-Dive (400-500 words)
5. Section 3: How It Works (500-600 words)
6. Section 4: Benefits (600-700 words)
7. Section 5: Options/Types (400-500 words)
8. Section 6: Cost/Pricing (400-500 words)
9. Section 7: Implementation (300-400 words)
10. Section 8: How NextPhone Solves This (200-300 words)
11. FAQ Section (300-500 words, 5-7 questions)
12. Conclusion (100-150 words)

**Total:** ~3,000-3,500 words

**Create detailed outline for ALL sections** following template above.

## Step 2.5: FAQ Section Planning (10 mins)

**Goal:** Cover remaining questions for SEO + user value

**Select 5-7 FAQ Questions** from Phase 1 research:
- Google's "People Also Ask" questions
- Reddit/Quora questions
- Questions NOT fully covered in main content

**Criteria:**
1. Not repetitive (avoid repeating main content)
2. High search volume (check Google PAA)
3. Address objections (cost, accuracy, implementation)
4. Quick to answer (2-3 sentences each)
5. Natural phrasing (how real people ask)

- Question (as H3, natural phrasing)
