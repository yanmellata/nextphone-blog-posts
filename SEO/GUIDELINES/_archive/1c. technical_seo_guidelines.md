1. GPT GENERATED

Technical SEO Guidelines for GetNextPhone.com

These ensure Google can crawl, index, render, and rank your blog posts properly. They also prevent issues that commonly kill SEO performance (slow pages, duplicate content, lack of schema, wrong URLs, etc.).

1. URL Structure

Use short, clean, human-readable URLs
Example:
/blog/ai-receptionist-benefits
NOT:
/blog/2025/11/17/how-an-ai-receptionist-can-help-your-small-business/

Always include the primary keyword in the slug.

Use lowercase, hyphens, no underscores.

2. Indexing & Crawlability

Every blog post should be indexable by default.

<meta name="robots" content="index, follow">

Confirm the URL appears in the XML sitemap within 24h of publishing.

Ensure you are not accidentally blocking Google:

robots.txt should not block /blog/

Avoid thin/duplicate pages. Remove or noindex:

Tag pages

Category pages with 0 posts

Pagination pages (optional)

3. Site Speed (Core Web Vitals)

Google heavily ranks based on speed and UX.

Requirements:

LCP < 2.5s

CLS < 0.1

INP < 200ms

Optimize by:

Compress all images via WebP.

Preload main fonts.

Lazy-load all images below the fold.

Minify JS/CSS.

Remove unused CSS (Tailwind + PurgeCSS if using).

Avoid heavy 3rd-party scripts.

Serve pages via a CDN (Vercel/Cloudflare).

4. Mobile Optimization

Everything must be fully responsive.

No horizontal scroll.

No text smaller than 14px.

Click targets min 44px.

Google tests with Mobile-First Indexing — mobile performance affects ALL ranking.

5. Internal Linking Rules

Every blog post must contain:

3–5 internal links to relevant pages

Product features

Pricing

Other blogs

Anchor text must be descriptive.

Use: AI receptionist for small business

Avoid: Click here

Link OLD posts → NEW posts whenever new content drops.

6. External Links

Google trusts pages that cite real sources:

Add 2–4 authoritative outbound links
(Harvard Business Review, Gartner, Statista, government data, etc.)

Always open external links in new tab.

This boosts E-E-A-T and avoids looking AI-generated.

7. Schema Markup (VERY important)

Add JSON-LD schema for:

BlogPosting

For each article:

@type: BlogPosting

headline

description

datePublished

dateModified

author

image

publisher

mainEntityOfPage

Breadcrumb Schema

Improves CTR and sitelinks.

Organization Schema

For GetNextPhone as the brand.

This is one of the highest-ROI technical SEO steps.

8. Canonical Tags

Every blog URL must contain a self-referencing canonical:

<link rel="canonical" href="https://getnextphone.com/blog/post-slug">


Prevents duplicate-content penalties if posts are shared or accidentally copied.

9. Image SEO

For every image in a post:

Filename must contain the keyword
ai-receptionist-phone-system.webp

Add alt text that describes what’s in the image.

Use WebP format.

Compress under 200 KB.

Add width/height attributes.

Images should never delay LCP.

10. Titles & Meta Descriptions

Even though this is “content,” meta tags are technical SEO.

Title tag rules:

50–60 characters

Include primary keyword

Clear, direct, no clickbait

Meta description:

140–160 characters

Include keyword

Summarize value

If the writer doesn’t deliver one, auto-generate using a fixed template.

11. H1–H3 Structure

Technical requirement:

ONE H1 only (the headline)

H2 for main sections

H3 for subsections

No skipping hierarchy (H1 → H3 is wrong)

Google uses these to understand topical hierarchy.

12. Avoid Technical SEO Errors That Kill Rankings

Never allow:

Duplicate H1s

Missing alt text

Render-blocking JS

Slow LCP above the fold

Links to 404 pages

Internal redirects (link to final URL, not redirects)

13. Structured Content Templates

To avoid AI-looking content, enforce predictable human structure:

Intro (3–5 sentences)

Table of contents (auto-generated)

3–6 H2 sections

Lists/steps/examples

FAQ Section (5–7 Qs)
→ Add FAQ schema where possible

Conclusion + CTA

This structure significantly increases dwell time and rankings.

14. Logging & Monitoring

Use Google Search Console to monitor indexing, errors, and CTR.

Use Google Analytics 4 to track scroll depth, time on page, bounce rate.

Use Ahrefs / SEMrush to monitor:

Keyword ranking movement

Backlinks

Content opportunities

Perform quarterly content audits.

15. Sitemap Management

Ensure /sitemap.xml automatically updates on every new post.

Submit it once to Search Console.

OPTIONAL: break into categories when you hit 100+ posts:

/sitemap-blog.xml

/sitemap-pages.xml

16. Safe Use of AI + Anti-Detection

Even though Google doesn’t penalize AI outright, your posts must NOT trigger low-quality signals.

Technical rules to avoid AI flags:

No repetitive sentence structures (LLM signature).

No filler phrases.

No paragraphs >150 words.

Use synonyms + varied language.

Include human elements: examples, case studies, data.

Written by humans → edited by humans → published by humans.

Also add:

<meta name="author" content="Full Name">


Avoid “Author: AI” anywhere.

17. Avoid Certain WordPress/Website Builder Traps

If using WP, Webflow, Next.js, or others:

Disable auto-indexing of:

/tag/

/author/

/feed/

Ensure blog pagination has:

Canonical tags

Noindex on page 2+

Disable duplicate archives.
