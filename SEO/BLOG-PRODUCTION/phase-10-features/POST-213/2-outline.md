### SECTION 3: Geographic Routing Methods

**H2 Title:** Geographic Routing Methods: ZIP Code, Area Code, and Caller ID

**Purpose:** Explain technical routing options in accessible language
**Word Count Target:** 300-350 words

**H3 Subsections:**
1. ZIP Code Routing (Most Accurate)
2. Area Code Routing (Automatic)
3. Caller ID Routing (Silent)
4. IVR Menu Selection (Manual Backup)

**Key Points to Cover:**
- ZIP code routing: Caller enters ZIP, system routes to franchise serving that territory (most accurate)
- Area code routing: Uses caller's phone number area code to determine location (automatic, less accurate)
- Caller ID routing: Silent detection based on caller's phone number (seamless, moderate accuracy)
- IVR menu: "Press 1 for Denver, Press 2 for Phoenix" (manual backup when automatic fails)
- Hybrid approach: Area code auto-route + ZIP verification for boundary cases

### SECTION 10: FAQ

**H2 Title:** Frequently Asked Questions

**Purpose:** Address remaining objections and questions
**Word Count Target:** 250-300 words (6 questions × ~45 words each)

**FAQ Questions:**

1. **How do I handle calls from customers on the boundary between two franchise territories?**
   - Answer: Set clear routing rules in advance. Options: Route to nearest location by distance, route to franchisee who opened first in that market, or route to larger territory. Document this in franchise agreement to prevent conflicts. Most systems allow you to assign boundary ZIP codes to one specific franchise.

2. **Can franchisees customize their location's greeting without breaking brand standards?**
   - Answer: Yes, with template-based greetings. Corporate sets required brand phrases (locked), franchisees customize location details like hours and service area (unlocked). Centralized dashboard lets corporate review and approve all changes before they go live.

3. **What happens when a franchise location closes or sells?**
   - Answer: Immediately update routing to remove that location. Calls from that territory should route to nearest remaining franchise or corporate answering service. Most cloud systems let you make this change in minutes. Traditional PBX systems require IT reprogramming.

4. **How much does multi-location call routing cost for a 10-location franchise?**
   - Answer: Costs vary widely. Traditional PBX: $50,000 setup + $2,000/month. Cloud VoIP: $0 setup + $900/month (assuming 3 users per location at $30/user). NextPhone: $0 setup + $199/month for unlimited locations. Savings increase as you add more franchise locations.

5. **Do I need IT staff to set up multi-location routing?**
   - Answer: Not with modern cloud systems. NextPhone and similar platforms offer web dashboards where you configure routing rules, assign ZIP codes, and set up locations without coding. Traditional PBX systems do require IT staff or vendor installation.

6. **How do emergency calls get routed across territories?**
   - Answer: Use emergency keyword detection. When caller says "emergency," "urgent," or "ASAP," system overrides territory routing and sends call to corporate on-call line or nearest available technician regardless of territory. This ensures emergencies get immediate response.
